Q-2

        let count = 0;
(function () {
  if (count === 0) {
    let count = 1;
    console.log(count); // What is logged?
  }
  console.log(count); // What is logged?
})();
    
Output:
        1
        0

the variable count is declared and initialized with a value of 0 outside of the immediately invoked function expression. Inside the function, a new variable named count is declared using the let keyword and initialized with a value of 1. This new variable count is block scoped and does not affect the outer count variable. When console.log(count) is called inside the if block, it will log the value of the inner count variable, which is 1. Outside the if block, when console.log(count) is called again, it will log the value of the outer count variable, which is still 0. The inner count variable does not affect the outer variable due to the concept of variable scope.

Q-3

"for (var i = 0; i < 3; i++) {
        setTimeout(function log() {
          console.log(i); // What is logged?
        }, 1000);
      }"
Output:
        3
        3
        3

In the code, a for loop is used to iterate from 0 to 2. Inside the loop, a setTimeout function is called with a delay of 1000 milliseconds, and it logs the value of i to the console. However, since var is used to declare the variable i, it has function-level scope, not block-level scope. This means that the variable i is shared among all iterations of the loop and is not scoped to the individual iterations. After the for loop finishes executing, the value of i will be 3. Therefore, when the setTimeout callbacks are executed after the delay, they will log the value of i, which is 3 for all of them.

function outerfunction() {

        }
      
        return increaseCounter; 
      }
      
     let counterFunc = outerfunction();
      
      counterFunc(); 
      counterFunc(); 
      counterFunc(); 
      counterFunc(); 
      counterFunc(); 
Output:
        1
        2
        3
        4
        5

In this example, the outer function creates an outer scope where the counter variable is defined and initialized with 0. The inner function increaseCounter has access to the counter variable in the outer scope and increments it by 1 every time it is called. The increaseCounter function is returned from the outer function. By calling outerfunction(), we create an instance of the counter function and assign it to the variable counterFunc. Then, we can call counterFunc() multiple times to increase the counter and log its updated value to the console.

        var x = (function () {
          var a = 12;
          return function () {
            alert(a);
          };
        })();
        x();"
Output
            12

In the code, there are two variables declared a and x. The variable a is declared and initialized with a value of 10. The variable x is declared and assigned the result of an function. Inside the function, a new variable a is declared and initialized with a value of 12. Then, an anonymous function is returned from the function. Finally, you call the function x(), which invokes the returned function from the parent function. This function references the variable a from its local scope, which is the inner a variable with a value of 12. It then alerts the value of a, which is 12. As a result, when we run this code, an alert dialog will appear with the value 12.
